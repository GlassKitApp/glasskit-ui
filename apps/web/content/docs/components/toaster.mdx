## When to use

Mount `<Toaster />` once at your app root, then fire from anywhere with `toast()`; it owns the queue, stacking, and auto-dismiss. A toast is fire-and-forget by default (a one-line confirmation that auto-dismisses); make it interactive by giving it an `action` (its button is D-pad focusable) plus `duration: Infinity` so it persists until pinched. For a persistent item the wearer must work through, use a [NotificationCard](/docs/blocks/notification-card) instead.
